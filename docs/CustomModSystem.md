# カスタムシステムの制作方法(Javaドキュメント)
JavaのコードでのMOD開発ではJavaを知っておく必要がある。
このため、Yuyutoが後々超規模なMODを大量生産した時に迷わないためにドキュメントを自分の言葉で制作する。
# 用語集

- メソッド：関数のこと。
- データ：変数のこと。
- クラス:設計図のこと。たとえば家を建てるときの設計図のようなもの。設計図には以下が書かれています：
  * 家の中の部屋（＝**変数/プロパティ**）
  * 家でできること（＝**メソッド/関数**）
例：
```java
public class CustomSystem {
    private int number;      // 部屋（箱）＝numberという名前の箱
    public void exp1func(){} // 家でできること＝exp1funcという行動
}
```
* この時点では「箱はまだ建っていない」状態です。

- オブジェクト:クラスから作った実際の箱（家）`new CustomSystem()` が「設計図から家を作る」操作に当たる。家ができると、中の部屋に物を入れたり、行動（メソッド）を呼んだりできる。

イメージ：

```
CustomSystem クラス（設計図）
        │
        ▼ new
obj  ┌─────────────┐
     │ number = 0  │  ← インスタンス変数
     │ exp1func()  │  ← メソッド
     └─────────────┘
```

1. **`CustomSystem`**

   * 「どの設計図の箱を作るか」
   * クラス名を指定している

2. **`obj`**

   * 「作った箱の名前」
   * 変数の一種で、このオブジェクトを指すラベル

3. **`new CustomSystem()`**

   * 実際に箱（オブジェクト）を作る操作
   * 作った箱の中には `number` という部屋や `exp1func()` という行動がセットされる

> つまり全体で「CustomSystemという設計図から箱を作って、それを obj というラベルで扱います」という意味.
例：
```java
CustomSystem a = new CustomSystem();
CustomSystem b = new CustomSystem();

a.number = 100; // aの箱だけ変わる
b.number = 50;  // bの箱だけ変わる
```

* これが **オブジェクト指向の核心**：「同じ設計図から独立した箱を作って、それぞれに値や機能を持たせる」こと
イメージ:
```
設計図: CustomSystem
        │
        │ new
        ▼
obj  ┌─────────────┐      obj2 ┌─────────────┐
     │ number = 0  │            │ number = 0  │
     │ exp1func()  │            │ exp1func()  │
     └─────────────┘            └─────────────┘
```
* `obj` と `obj2` は別々の箱
* `number` を変えても互いに影響しない
* ただし **static変数** は全オブジェクトで共通（箱の外にある共通のものとして扱われる）

# コマンドスコープ
|コマンドスコープ名|タイプ|説明|注意|書き方|
|------------------|------|----|----|
|package|パッケージ住所|当該クラスを他のクラスで使う場合に使われるパッケージ住所を記載|必ず/javaのディレクトリからドット(.)で一致させること|`package com.yuyuto.infinitymax.systems;`|
|import|パッケージ住所|指定したディレクトリのクラスを読み込んで利用可能にする。依存関係を解消する大事なコマンド。|必ず/javaのディレクトリからドット(.)で一致させること|`import com.yuyuto.infinitymax.systems;`|
|public|ラベル|クラス内の全メソッド(関数)で使えるメソッド・データだということを指定するためにつける。|付ける場所に気をつけよう|`public static void init(){}`|
|private|ラベル|そのメソッド内だけしか使えないことを示すもの|付ける場所に気をつけよう|`private static int a`|
